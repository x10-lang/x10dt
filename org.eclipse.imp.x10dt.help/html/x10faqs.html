How to show compiler options...

$ x10c -- -help
will print details of the various compiler options.

How to show where dynamic checks are...

the compilers shows me how many dynamic checks it inserted: 

$ x10c DataFlowExample.x10 
x10c: 7 dynamically checked calls or field accesses.
Is there a way to tell, where exactly those dynamic checks are? Maybe they could be made static with some code changes.
compile with 
-VERBOSE_CALLS=true
This will print details of all inserted dynamic casts.

How to show compile time errors...

As it is, the compiler inserts a dynamic check for you, which then fails at runtime. If you ran the compiler with the -STATIC_CALLS flag, you would have gotten a compile-time error.

How to use ! ...

Every object has a place, that's the point of the PGAS paradigm. You can access fields and methods of the object only in it's home place. Only "global" methods and fields can be accessed everywhere.

You should use "!" to make sure you don't have ClassCastException for home constraints. When you compiled your program, you probably saw the warning: ZZZ dynamic checks added. If you compile with the parameter -STATIC_CALLS then the compiler will give you errors instead of warnings, and you will know what to mark as "!".

Struct initialization...

For structs you do not need to use "new". Simply write: 

val x = Pair[Int,Int](3,4);
We have a proposal that would allow also using "new" for creating a struct (which will be more comfortable for Java users).
