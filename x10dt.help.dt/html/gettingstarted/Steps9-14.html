Remote Launching and Debugging Basics

X10DT allows you to compile and run your programs on different machines with the C++ back-end.
  • The X10 Platform Configuration and Remote Launching
  • Debugger Basics

The X10 Platform Configuration and Remote Launching

To configure how your program will compile and run on different machines, open the X10 Platform Configuration located at the root directory of your C++-backend project. The platform configuration has 2 tabs: Connection and Communication Interface and C++ Compilation and Linking. The communication interface allows you to select the type of interface and to configure it. The second tab allows you to specify the configuration of C++ compilation and linking.

To setup a remote connection, go to the Connection and Communication Interface tab and select Remote Connection. Click on Add and type the name of the new remote connection in the space inside the table under Name. Enter the host name and the user name, as well as the password. Then hit the Validate button next to the table. If your settings are correct a green check mark will appear next to the name of the connection.

Change the Connection Type to Open MPI and keep the default settings that appear.



Switch to the C++ Compilation and Linking tab. Specify the output folder in the home directory for the account you have chosen, and name your output folder using your own name. This is to ensure that different participants in the tutorial won't share output directories accidentally. For example, if you chose tutorial01 for your account, then your output folder would be:

/home/tutorial01/JohnDoe

In the X10 Distribution field, enter: /opt/x10-2.1.0.1_ompi. This will be automatically carried to the PGAS distribution field.
Be sure that the checkbox Use MPI Library for Debugger under Compilation and Linking is checked.



Finally, hit the Validate Configuration button located at the top of the platform configuration. If all your settings are correct, this button will turn into a smiley face. If you don't get a smiley face, go back to the 2 tabs and fix your settings according to the instructions above.



After your platform configuration is set, you need to run the program once. To do so, go to:

Run -> Run Configurations...

Double click on X10 Application (C++ back-end) to create a new Run Configuration. In the Application tab, enter the desired project name in the X10 Project field, and the name of the main type in the Main class field. You may use the Search... button to get the main type automatically.



In the Communication Interface tab, in the Basic Open MPI section, enter the desired number of processes (needs to be >0). Check the Host file check box and enter (with tutotrial01 replaced with your account):

/home/tutorial01/.rhosts



Click on the Advanced Open MPI button, and check the check boxes labeled: Use default arguments, and Use default parameters. Occasionally, you may need to hit Apply when making these changes because the settings may be lost as you navigate through the tabs (a shortcoming that will be fixed in a future release).



When you are finished with the Run Configurations settings, and no tab shows any errors, hit Apply to save, and then hit Run. This will open the Parallel Runtime view, where you will see the output of your program when it's finished.

If this step fails for you, go to the Resources Manager pane in the Parallel Runtime perspective. Right click on the resource manager corresponding to your project, and select Stop Resource Manager. Click OK in the popup window. Then right click again on the resource manager, and select Edit Resource Manager. In the window that appears, be sure that the check box SSH Port Forwarding is checked and click Finish. Right click again on the resource manager and select Start Resource Manager. Then run your program again by going back to the run configurations (Run -> Run Configurations...) and selecting the run configuration you created above.



Debugger Basics

To start working with the debugger, go to:

Run -> Debug Configurations...

Double click on Parallel Applications to create a new debug configuration. Under the Resources tab, select the resource manager corresponding to your project. By default, this resource manager has the same name as your project. Select the number of processes desired (> 0). Check the Host file check box and enter (with tutotrial01 replaced with your account):

/home/tutorial01/.rhosts



Click on the Advanced Open MPI button, and check the check boxes labeled: Use default arguments, and Use default parameters.



In the Application tab, enter the name of the X10 project to debug in the Parallel Project field. Click the Browse button next to the Application Program field to choose the name of the application you wish to debug.



In the Debugger tab, select SDM as the Debugger and x10-gdb-mi for the Debugger backend.

For the Path to the debugger executable enter:

/opt/PTP_Debugger/1111/org.eclipse.ptp.linux.x86/bin/sdm



When you are finished with the Debug Configurations settings, and no tab shows any errors, hit Apply to save, and then hit Debug. This will open the Parallel Debug perspective, where you will be able to step through your program and debug it. For more information on how to use the debugger see here.

If this step fails for you and you don't see the Parallel Debug perspective, go to the Resources Manager pane in the Parallel Runtime perspective. Right click on the resource manager corresponding to your project, and select Stop Resource Manager. Click OK in the popup window. Then right click again on the resource manager, and select Edit Resource Manager. In the window that appears, be sure that the check box SSH Port Forwarding is checked and click Finish. Right click again on the resource manager and select Start Resource Manager. Then debug your program again by going back to the debug configurations (Run -> Debug Configurations...) and selecting the debug configuration you created above.
