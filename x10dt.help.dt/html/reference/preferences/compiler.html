Conservative build

This is a new compiler option "Conservative Build", which is turned on by default. When turned on, this option causes each file that has no generated code in a project to be re-submitted for compilation on every build. This ensures that all the files that need to be recompiled are indeed compiled when the user saves in an editor, and overcomes some shortcomings of the compiler which make it difficult to compute dependencies between files. However, this feature is overly conservative, and thus can cause the builder to be slower. To make builds faster, this option may be turned off. In this case, the user may occasionally observe stale compilation errors. This of course can be easily remedied by doing a clean every now and then.
