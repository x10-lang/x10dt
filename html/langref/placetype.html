14  Place types

PlaceTypeSpecifier ::= ! PlaceType
PlaceType ::=  any  |  current  |  Expression
All X10 reference classes extend the class x10.lang.Ref: 
package x10.lang;
public class Ref(place loc)  ...
Because of the importance of places in the X10 design, special syntactic support is provided for deptypes involving places.
We now consider the expansions of deptypes with place information.
Unless a deptype T (whose base is a reference type) has an ! suffix, the constraint for T is implicitly assumed to contain the clause self.loc==here. 
C( : c) => C(: self.loc==here && c )
The expansions for a deptype with an "!" suffix are: 
C( : c)!  => C(: c )  // no self.loc clause.
C( : c)!p => C(: self.loc==p && c )
Static Semantics Rule: It is a compile time error for the !-annotation to be used for deptypes whose base type does not extend x10.lang.Ref.
