X10DT - New and Noteworthy

X10DT 1.7.4.0 supports the X10 language version 1.7, on the Java back-end version of X10. It includes a syntax-highlighting editor, Outline view, run/launch capabilities, and user preferences.

X10 Editor

Installation

 Use the Eclipse Update Manager to install X10DT. See installation instructions on x10-lang.org . Everything you need is in one update site.

Welcome

 When you first install X10DT and restart Eclipse, the welcome page shows with the X10 information highlighted (yellow indicates a just-installed feature of Eclipse). Click here to read about X10DT usage.

To continue to the X10DT workbench, close the welcome tab, or click on the curved-arrow "Go to workbench" icon at the top. Then switch to the X10 perspective (Window > Open Perspective > Other... and select X10)

New Project Wizard

 Create a new X10 project with the New Project Wizard. By default the "Create a sample "Hello World" X10 application is selected.

The X10 workbench is filled out with information from your new project.

X10 Editor

X10 Editor basics
  • Editor recognizes X10 1.7 syntax; keywords, comments, strings, etc are colored appropriately.
  • Hyperlinking of e.g. a method or variable navigates to the definition of that element. Hyperlinking works via Cntrl-left Click (on Mac OSX: Command-click)
  • Outline view tracks contents of editor.
  • Error markers show compilation problems

Refactoring
  • Rename refactoring: currently supports renaming of local entities (select member element and bring up context menu: Refactor > Rename)
  • Several refactorings specifically for increased concurrency:
    • Loop Flat Parallelization
    • Mark Finish Context
    • Extract Async
    • Extract Async in Finish Scope

    Preview window lets you confirm the changes to be made

Run/Launch
  • Run Configurations are accessed via the Run menu

  • Program Arguments and VM command line arguments may be specified on the Arguments tab.

Preferences
  • Font selection (currently not operational)
  • Default runtime location (allows specification of a different X10 runtime library)
  • Checkboxes allow selection or deselection of
    • Bad place runtime check
    • Loop optimizations
    • Array optimizations
    • Permit asserts
    • Echo compile arguments to console - if selected, prints the compile arguments to the x10 compiler in an Eclipse console on each compile.
    • Number of X10 places
    • Additional compiler options
