I installed just the X10DT and the CDT "New C/C++ Project" wizard dies. Why?

Answer: The X10DT uses PTP, which in turn requires just a part of the CDT as a prerequisite. Due to a problem with the way the CDT is packaged, the wizard appears in the CDT's menus, but the wizard's implementation is not actually available. To fix this problem, you need to install the CDT (see below).

How can I use both the X10DT and the CDT in the same Eclipse install?

Answer: Install the CDT first from any of the usual Eclipse update/discovery sites, and then install the X10DT.

How can I change the font in the X10DT editor?

Answer: You can change it in the Preferences in the following page. General > Appearance > Colors and Fonts. Then under Basic folder, click on Text Font and then click the Edit button.

Memory consumption

Answer: Memory consumption by the editor and the build process has been improved since previous releases. It is still a work in progress. You may encounter either two memory-related errors:

  • Java Heap Space OutOfMemory: You can increase the maximum heap size by specifying the -Xmx (Sun VM) VM option in the eclipse.ini in your Eclipse installation. For more details, see the Eclipse online help.
  • Java PermGen OutOfMemory: You can increase the PermGen area's size by specifying the -XX:MaxPermSize (Sun VM) in the eclipse.ini file.
Additionally, it may be useful to reduce the number of simultaneously opened editors.

Builder performance

Answer: The current build mechanism is somewhat conservative in determining the set of files to be recompiled upon each save. This will cause longer build times under certain circumstances.

We've provided a building option, "Conservative Build" - turned on by default, which if turned off (see X10 Preferences) can result in shorter build times. However the shorter build time comes at the expense of incorrect behavior for certain X10 source programs.
