What's New in 2.4.3

Here are descriptions of some of the more interesting or significant changes made in X10 and X10DT for the 2.4.3 release. They are grouped into:

  • X10 Language
  • X10 Development Toolkit
  • Managed X10
  • Native X10
  • Additional Notes
X10 Language
Elimination of busy-waiting in X10 Runtime By default, if there are no runnable activities in a Place the X10 runtime now blocks all idle worker threads (thus consuming no CPU cycles). To obtain the previous behavior (polling loop), X10_BUSY_WAITING=true in the environment.
X10 Development Toolkit
Debugger for Managed X10

X10DT now supports basic source-level debugging functionality for Managed X10. Specifically, Managed X10 applications are now compiled with JSR-045 SMAP information which enables a mapping between source lines in the X10 source and compiled Java classes. This allows line-oriented source-level debugging actions (breakpoints, stepping, etc) to be supported for Managed X10 applications.

However on Linux, to get source-level breakpoints to work you must not use the IBM JVM. The IBM JVM does not always properly process the JSR-045 SMAP information for breakpoints. Instead use the OpenJDK or Oracle JVM (by ensuring that the Oracle or OpenJDK JVM is the default Java in your PATH when you invoke X10DT).
