What's New in 2.4.2

Here are descriptions of some of the more interesting or significant changes made in X10 and X10DT for the 2.4.2 release. They are grouped into:

  • X10 Language
  • X10 Development Toolkit
  • Managed X10
  • Native X10
X10 Language
Resilient X10 A technology preview of Resilient X10 is including in this release. Resilient X10 (described in an upcoming PPoPP'14 paper ) adds the ability for X10 programs to continue execution even if one or more places fail. For details on how to use Resilient X10, please see the notes and example programs in samples/resiliency.
X10 Development Toolkit
Upgrade to Eclipse 4.3 and PTP 7 X10DT is now based on Eclipse 4.3 and PTP 7.
Support for multi-place Managed X10 on Windows X10DT supports the new pure-Java launcher for Managed X10, enabling multi-place X10 executions to be launched from X10DT on Windows.
Managed X10
Multi-Place Managed X10 on Windows We have completed a new pure-Java launcher to complement the pure-Java implementation of X10RT (JavaSockets) from the X10 2.4.1 release. As a result it is now possible to execute multi-place Managed X10 programs on a single Windows host using X10DT without needing to install cygwin. Cygwin is still required on Windows to build X10 from source, to run Native X10 programs and to use the X10 command line tools.
Native X10
Retirement of BlueGene/P Due to the retirement of BlueGene/P installations, X10 support for BlueGene/P was removed in X10 2.4.2. X10 will continue to support the BlueGene/Q platform.
X10 on CUDA devices A regression in X10 support for CUDA introduced in X10 2.4.0 was fixed in 2.4.2. With X10 2.4.2, all CUDA sample programs execute successfully using the latest (5.5) CUDA toolkit release.
