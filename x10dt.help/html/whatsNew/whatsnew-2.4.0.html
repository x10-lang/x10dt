What's New in 2.4.0

Here are descriptions of some of the more interesting or significant changes made in X10 and X10DT for the 2.4.0 release.

X10 Language
Emphasis on 64-bit computing With the X10 2.4.0 release, significant changes were made to the X10 language specification and X10 core class libraries to better support 64-bit computing.
  • All Rail, Array, Point, Region, etc. classes use Long (64 bit) indices instead of Int (32 bit).
  • The type of an unqualified integral literal (e.g., 3) is a Long instead of an Int
  • Numerous class library APIs related to indexing and collection size were changes from Int to Long
Higher performance default Arrays
  • A high-performance x10.lang.Rail type was introduced to obtain the best possible performance for the common case of a one-dimensional N-element array with indices from 0 to N-1.
  • The existing Region-based Array classes were moved to a new package, x10.regionarray, which is not automatically imported.
  • A set of new high-performance multi-dimensional arrays for the common case of dense, zero-indexed arrays can be found in the x10.array package. The next release of X10 will include additional distributed variants of these high-performance arrays.
Native X10
Support for BlueGene/Q Native X10 now supports the BlueGene/Q platform.
