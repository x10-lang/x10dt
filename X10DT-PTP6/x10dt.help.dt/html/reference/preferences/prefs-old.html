To modify how your X10DT operates, bring up the X10 preferences page.

On most platforms, Window > Preferences.

On Mac, Eclipse > Preferences.

Generic preferences concepts: The X10DT preferences show when values have been modified, and show at what level the preferences are set - See the main X10 preferences page, shown above, for more information. As you change a preference setting, for example, its color changes to red indicating its value has changed.

Some of the preferences options are described below.

X10 general Preferences

On the main X10 preference page, you can set the tab width, and whether or not to use spaces for tabs. (See above.)

Compiler Preferences



Conservative Build

This is a new compiler option "Conservative Build", which is turned on by default. When turned on, this option causes each file that has no generated code in a project to be re-submitted for compilation on every build. This ensures that all the files that need to be recompiled are indeed compiled when the user saves in an editor, and overcomes some shortcomings of the compiler which make it difficult to compute dependencies between files. However, this feature is overly conservative, and thus can cause the builder to be slower. To make builds faster, this option may be turned off. In this case, the user may occasionally observe stale compilation errors. This of course can be easily remedied by doing a clean every now and then.

Echo Compile Arguments to Console

To see the x10 compiler arguments, check this option.

Then, each time you build, the compile options will appear in the Console view.
